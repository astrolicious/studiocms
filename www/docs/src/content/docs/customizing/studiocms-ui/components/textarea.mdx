A simple textarea component with support for easy labels and placeholders.

## Usage

<Tabs>
  <TabItem label="Preview">
    <PreviewCard>
      <Textarea label="Label" placeholder="Placeholder" />
    </PreviewCard>
  </TabItem>
  <TabItem label="Code">
    ```astro
    ---
    import { Textarea } from '@studiocms/ui/components/Button';
    ---

    <Textarea label="Label" placeholder="Placeholder" />
    ```
  </TabItem>
</Tabs>

### Disabled

You can disable the textarea altogether by using the `disabled` prop.

<Tabs>
  <TabItem label="Preview">
    <PreviewCard>
      <Textarea label="Label" placeholder="Placeholder" disabled />
    </PreviewCard>
  </TabItem>
  <TabItem label="Code">
    ```astro "disabled"
    ---
    import { Textarea } from '@studiocms/ui/components/Button';
    ---

    <Textarea label="Label" placeholder="Placeholder" disabled />
    ```
  </TabItem>
</Tabs>

### Form Support

Textarea have full form support. You can use the `isRequired` and `name` attributes as needed. 

The first one does what it says it does, while the latter sets the `name` attribute so you can access the textarea in, for example,
the `FormData` returned by a submission. Here's an example showing how to set both of them:

<Tabs>
  <TabItem label="Preview">
    <PreviewCard>
      <Textarea label="Label" placeholder="Placeholder" isRequired name="example-textarea" />
    </PreviewCard>
  </TabItem>
  <TabItem label="Code">
    ```astro "isRequired" "name=\"example-textarea\""
    ---
    import { Textarea } from '@studiocms/ui/components/Button';
    ---

    <Textarea label="Label" placeholder="Placeholder" isRequired name="example-textarea"/>
    ```
  </TabItem>
</Tabs>

:::note
If no name is set, a random one is generated for the component.
:::
